# 客服功能接口文档

## 一、用户角色管理

### 1.1 获取用户信息接口改造

**接口地址**: `GET /api/user/info`

**响应参数扩展**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | string | 是 | 用户ID |
| account | string | 是 | 用户账号 |
| role | string | 是 | 用户角色（`user` - 普通用户，`customer_service` - 客服） |

**响应示例**:

```json
{
  "code": 200,
  "data": {
    "userId": "user123",
    "account": "zhangsan",
    "name": "张三",
    "avatar": "/uploads/avatar/xxx.png",
    "balance": 100.00,
    "role": "customer_service",
    "phone": "13800138000",
    "createTime": "2024-01-01T00:00:00.000Z"
  }
}
```

**说明**: 后端需要在用户信息接口的返回数据中添加 `role` 字段，用于前端判断用户角色并跳转到对应页面。

## 二、登录流程改造

### 2.1 登录成功后跳转逻辑

前端登录成功后，会根据返回的用户信息中的 `role` 字段进行页面跳转：

| 角色 | 跳转页面 |
|------|----------|
| `user` | `/pages/index/index`（首页） |
| `customer_service` | `/pages/customer-service/index`（客服工作台） |

**前端实现位置**: `src/pages/login/index.vue`

```javascript
// 根据用户角色跳转到不同页面
if (userInfo.data.role === 'customer_service') {
  // 客服跳转到客服工作台
  uni.reLaunch({
    url: "/pages/customer-service/index",
  });
} else {
  // 普通用户跳转到首页
  uni.reLaunch({
    url: "/pages/index/index",
  });
}
```

## 三、客服相关API接口

### 3.1 获取客服会话列表

**接口地址**: `GET /api/customer-service/sessions`

**请求头**: 需要携带用户登录token

**响应参数**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 会话ID |
| customerId | string | 是 | 客户ID |
| customerName | string | 是 | 客户名称 |
| customerAvatar | string | 否 | 客户头像 |
| lastMessage | string | 是 | 最后一条消息内容 |
| lastMessageTime | string | 是 | 最后一条消息时间（格式：YYYY-MM-DD HH:mm） |
| unreadCount | number | 是 | 未读消息数量 |

**响应示例**:

```json
{
  "code": 200,
  "data": [
    {
      "id": "session_001",
      "customerId": "user_001",
      "customerName": "张三",
      "customerAvatar": "/uploads/avatar/user001.png",
      "lastMessage": "您好，我有一个问题想咨询",
      "lastMessageTime": "2024-01-15 10:30",
      "unreadCount": 2
    },
    {
      "id": "session_002",
      "customerId": "user_002",
      "customerName": "李四",
      "customerAvatar": "/uploads/avatar/user002.png",
      "lastMessage": "谢谢，问题已经解决了",
      "lastMessageTime": "2024-01-15 09:15",
      "unreadCount": 0
    }
  ]
}
```

### 3.2 获取与客户的聊天记录

**接口地址**: `GET /api/customer-service/chat/:customerId`

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customerId | string | 是 | 客户ID |

**请求头**: 需要携带用户登录token

**响应参数**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customerInfo | object | 是 | 客户信息对象 |
| customerInfo.id | string | 是 | 客户ID |
| customerInfo.name | string | 是 | 客户名称 |
| customerInfo.avatar | string | 否 | 客户头像 |
| messages | array | 是 | 消息列表 |
| messages[].id | string | 是 | 消息ID |
| messages[].content | string | 是 | 消息内容 |
| messages[].messageType | string | 是 | 消息类型（`text` - 文本，`image` - 图片） |
| messages[].senderId | string | 是 | 发送者ID |
| messages[].receiverId | string | 是 | 接收者ID |
| messages[].createdAt | string | 是 | 消息创建时间 |
| messages[].isMine | boolean | 是 | 是否为当前用户发送的消息 |

**响应示例**:

```json
{
  "code": 200,
  "data": {
    "customerInfo": {
      "id": "user_001",
      "name": "张三",
      "avatar": "/uploads/avatar/user001.png"
    },
    "messages": [
      {
        "id": "msg_001",
        "content": "您好，我想咨询一下问题",
        "messageType": "text",
        "senderId": "user_001",
        "receiverId": "cs_001",
        "createdAt": "2024-01-15T10:25:00.000Z",
        "isMine": false
      },
      {
        "id": "msg_002",
        "content": "您好，请问有什么可以帮助您的？",
        "messageType": "text",
        "senderId": "cs_001",
        "receiverId": "user_001",
        "createdAt": "2024-01-15T10:26:00.000Z",
        "isMine": true
      }
    ]
  }
}
```

### 3.3 发送消息给客户

**接口地址**: `POST /api/customer-service/message/:customerId`

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customerId | string | 是 | 客户ID |

**请求头**: 需要携带用户登录token

**请求参数**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| text | string | 否 | 文本消息内容 |
| messageType | string | 是 | 消息类型（`text` - 文本，`image` - 图片） |
| image | string | 否 | 图片URL（当messageType为image时必填） |

**请求示例**:

```json
{
  "text": "您好，请问有什么可以帮助您的？",
  "messageType": "text"
}
```

或者发送图片：

```json
{
  "image": "/uploads/chat/image/xxx.png",
  "messageType": "image"
}
```

**响应参数**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 消息ID |
| status | string | 是 | 发送状态（`sent` - 已发送） |

**响应示例**:

```json
{
  "code": 200,
  "data": {
    "id": "msg_003",
    "status": "sent"
  }
}
```

### 3.4 标记会话为已读

**接口地址**: `PUT /api/customer-service/session/:sessionId/read`

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| sessionId | string | 是 | 会话ID |

**请求头**: 需要携带用户登录token

**响应参数**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| status | string | 是 | 操作状态（`success`） |

**响应示例**:

```json
{
  "code": 200,
  "data": {
    "status": "success"
  }
}
```

## 四、WebSocket消息格式

### 4.1 客服端接收客户消息

当客户发送消息给客服时，WebSocket推送的消息格式：

```json
{
  "type": "message_received",
  "messageId": "msg_004",
  "customerId": "user_001",
  "content": "这是客户发送的消息内容",
  "messageType": "text",
  "createdAt": "2024-01-15T10:30:00.000Z"
}
```

如果是图片消息：

```json
{
  "type": "message_received",
  "messageId": "msg_005",
  "customerId": "user_001",
  "content": "",
  "image": "/uploads/chat/image/xxx.png",
  "messageType": "image",
  "createdAt": "2024-01-15T10:31:00.000Z"
}
```

### 4.2 消息状态更新

```json
{
  "type": "message_sent",
  "messageId": "msg_003",
  "status": "delivered"
}
```

## 六、前端页面路由

### 6.1 客服工作台

**页面路径**: `/pages/customer-service/index`

**功能说明**:
- 显示所有客户会话列表
- 按最后消息时间排序
- 显示未读消息数量
- 点击会话进入聊天界面

### 6.2 客服聊天页面

**页面路径**: `/pages/chat/index?type=customer_service&customerId={customerId}`

**URL参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| type | string | 是 | 固定值 `customer_service` |
| customerId | string | 是 | 客户ID |

**功能说明**:
- 与指定客户进行实时聊天
- 支持发送文本消息
- 支持发送图片消息
- 客服头像显示为 `/static/client.png`

## 七、注意事项

1. **权限验证**: 所有客服相关接口需要在请求头中携带有效的用户登录token，后端需要验证用户角色为 `customer_service`。

2. **消息推送**: 建议使用WebSocket实现实时消息推送，当客户发送消息时主动推送给客服。

3. **会话管理**: 当客户首次发起咨询时，后端需要自动创建会话；当客服回复消息时，更新会话的最后消息时间和未读状态。

4. **图片存储**: 客服上传的图片建议存储到与普通聊天图片相同的目录，保持路径格式一致。

5. **错误处理**: 所有接口需要返回标准的错误码和错误信息，便于前端进行错误提示。

6. **消息分页**: 如果会话消息较多，建议支持分页加载，可在获取聊天记录接口中添加分页参数。
